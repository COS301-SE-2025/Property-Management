##  Core Features

### Contractor Management
- **Comprehensive Contractor Profiles** - Track contractor details, specializations, and project history
- **Performance Rating System** - Evaluate and rate contractor performance across multiple projects
- **Work Documentation** - Store proof of completed work with project mapping
- **Expense Integration** - Seamlessly integrate contractor costs into annual budgets

### Inventory Tracking & Management
- **Real-time Inventory Monitoring** - Track in-and-out flow of inventory items with precise usage mapping
- **Trustee Approval Workflow** - Implement approval processes for inventory usage requests
- **Project-Specific Usage** - Map inventory consumption to specific projects and contractors
- **Demand Forecasting** - AI-powered prediction of future inventory requirements

### Smart Quote Management
- **Multi-Quote Consolidation** - Collect and compare quotes from multiple contractors
- **Approval Workflow** - Streamlined trustee approval process for quotes and expenses
- **Cost Analysis** - Advanced analytics for quote comparison and decision-making

### AI-Powered Insights
- **Anomaly Detection** - Identify unusual maintenance costs and potential fraudulent quotes
- **Predictive Analytics** - Forecast maintenance schedules and budget requirements
- **Data-Driven Dashboard** - Comprehensive insights on cost trends and contractor reliability

##  Getting Started

### Prerequisites

- **Node.js** (v18+) and npm
- **Java** (v21+) and Maven/Gradle
- **PostgreSQL** (v13+)
- **Docker** and Docker Compose
- **AWS CLI** (configured)

### Quick Setup

```bash
# Clone the repository
git clone https://github.com/COS301-SE-2025/Property-Management.git
cd property-management

# Start with Docker Compose (recommended)
docker-compose up -d

# Or run components individually:

# Database setup
./scripts/setup-database.sh

# Backend setup
cd Backend/property-management
./gradlew bootRun

# Frontend setup
cd Frontend/Property-Manager
npm install
ng serve
```

### Environment Configuration

```bash
# Copy environment templates
cp backend/.env.example backend/.env
cp frontend/src/environments/environment.example.ts frontend/src/environments/environment.ts

# Configure your AWS credentials and database connections
```

Access the application at `http://localhost:4200`

## ğŸŒ³ Git Strategy

We implement a robust GitFlow branching strategy ensuring code quality and collaborative development:

### Branch Structure
- **`main`** - Production-ready, stable releases only
- **`dev`** - Integration branch for ongoing development
- **`frontend/*`** - All code concerning frontend features
- **`backend/*`** - All code concerning backend features


### Workflow Process
1. Create feature branches from `dev`
2. Implement changes with comprehensive tests
3. Submit pull requests with detailed descriptions
4. Code review process (minimum 2 approvals required)
5. Automated CI/CD pipeline validation
6. Merge to `dev` after approval
7. Regular releases from `dev` to `main`
